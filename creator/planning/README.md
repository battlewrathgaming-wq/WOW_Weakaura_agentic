# planning

`creator/`'s **first messy space** — the incubator. Where invent-side work is shaped *before* it's clean enough to be
part of `creator/` proper. Nothing here is settled; it's meant to be a bit messy.

## The earn-a-home model (Battlewrath)

Work incubates here. As a part gets **defined and proven**, it **graduates up into `creator/`** (or, if it turns out
mechanical, into the engine room — by nature/utility). This is how the engine crystallized: the work happened, and parts
earned their homes (`Fact_basis/sheets`, `contract`, `maps`, `Production/`). Same model.

**Self-dissolving by design.** As things graduate, `planning/` shrinks. When it empties, it has served its purpose and
gets **deleted** — a temporary mess, not a permanent room.

**Keep existing materials in place.** Do NOT migrate the current sprawl (`wa_index/`, `plane/`, the scattered scripts)
preemptively. Each earns its location when it's *defined*. No churn before a part is real.

## Currently staging

- **research** — external prior art for the classification tool: adtech signal platforms, knowledge-graph centrality,
  feature stores. Borrow proven architecture; don't reinvent.
- **settlement** — the migration plan: what graduates into `creator/` (or the engine), in what order, once earned.
- **the classification tool** — the signal platform: spell nodes + the `coa.triggeredBy` web + traffic (centrality) +
  lane buckets + the clear-vs-hard triage. Grounded (`operations/STATE.md`); design + build incubate here.

## The rule

Settled + proven → it belongs in a home (`creator/` or the engine), not here. `planning/` holds only what's still being
shaped. An empty `planning/` means the era's work has all earned its place — and the folder goes.
