# WHAT — what the project is

_A touchstone: stable, slow to move. **Locked 2026-07-14** (Battlewrath's layering, agreed). Read top-down: each layer
is the ground the next stands on. Changing this is a deliberate act, not an edit._

## Layer 0 — the posture (spirit)

An agentic, machine-leveraged way of working. AI/agents are creative and unlock projects one person couldn't reach
alone — and they drift, hallucinate, rush to please, and invent. So the whole system is built **with that awareness
inside it**. The collaboration posture is crystallized in the Aura-Project-Orchestration Overseer contract (care about
the project · challenge the model · preserve the Human's authority · don't wake Dev until the work is shaped).

## Layer 1 — why it's built this way (force correctness, don't trust perfection)

The pipeline exists to **constrain the agentic errors without being punitive to the workflow.** The gate, the
determinism, source-as-truth, nothing-authors-past-the-gate — these are the safeguards. **You don't have to trust
perfection; only that we've done the work to force correctness.** Creativity runs safely inside the contained spaces;
correctness is forced by structure, not vigilance.

## Layer 2 — the mission (community, at scale)

A community project, to serve the many. Scrape every ability; do the DBC work — so we can **mass-produce auras that
would otherwise be near-impossible.**

## Layer 3 — the approach (data-driven → bespoke → addons)

Produce WeakAuras from a **data-driven, classified** approach. Then leverage the same data for **complex, bespoke** aura
products. And, beyond auras, **World of Warcraft addons.**

## Layer 4 — the value (personal, with a free byproduct)

A lot of this **enriches Battlewrath's own experience of the game.** The community gets the byproduct **for free.**
Once the capability exists, there is no limit but **time and staging the work.**

---

_MVP sequencing (deferral, NOT exclusion): no shipped HUD / layout **yet** — users arrange their own UI, so the content
gets out first. Once it's out there, a later phase builds a **system / classification for HUD production** — the same
data-driven approach turned on layout (Layer 3 leverage). Content first; the HUD is a staged next, bounded by time, not
capability._
