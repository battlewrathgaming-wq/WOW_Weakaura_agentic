# Display

Everything here is meant to be opened directly in a browser - no server,
no build step. This folder is an index/pointer layer, not a copy: the
actual files still live under `Outputs/`, generated by the scripts in
`Scripts/` from `Input/` + the game client's Spell.dbc. Links below are
relative and will work from this file's location.

## Class ability sheets (all 21 classes, one file)

- [ability_sheets_all_classes.html](../Outputs/class_sheets/ability_sheets_all_classes.html) -
  class dropdown -> spec tabs -> Toolkit (first-class, pressable abilities)
  vs. Talents (passive/proc modifiers) lanes. Real Spell.dbc stats
  (cooldown, cast time, proc chance, duration, radius, school) on every
  card. No talent-relationship graph - this is the flat reference sheet,
  safe to hand out as-is. Built by
  `Scripts/build_class_ability_sheet.py`.

## DBC-sourced ability analysis (Necromancer only so far)

The talent-relationship pipeline - which talents own, trigger, teach,
modify, or compound on which abilities, first-class vs. second-order
classification, family/compounds detection. See `Docs/PIPELINE.md` section
9 for the full mechanism writeup.

- [necromancer_abilities_reference.html](../Outputs/dbc_preview/necromancer_abilities_reference.html) -
  every distinct ability the Necromancer tree touches, one anchor card per
  ability, every related talent listed underneath with an inline
  explanation of what it does. Built by
  `Scripts/build_abilities_reference_dbc.py`.
- [necromancer_talent_context.html](../Outputs/dbc_preview/necromancer_talent_context.html) -
  the same graph, tabbed per spec (Animation / Death / Rime / Class),
  abilities with zero applying talents for a spec dropped from that spec's
  view. Built by `Scripts/generate_talent_context_dbc.py`.
- [necromancer_ability_cards.html](../Outputs/dbc_preview/necromancer_ability_cards.html) -
  earliest prototype (one card per talent's own spell only). Superseded by
  the abilities reference above; kept for history.

## Production readouts (all 21 classes, prose-regex model)

The original, class-by-class flattened talent readouts - one file per
class, dark theme, per-class accent color, collapse/expand tree view.
Predates the DBC pipeline; still the most complete "read the whole tree"
view since it covers every class, not just Necromancer.

- `Outputs/readout_html/<class>_readout.html` - all 21 classes. Built by
  `Scripts/generate_readout_html.py`.

## Experimental: produces/nesting model (Necromancer only)

- [necromancer_readout_nested.html](../Outputs/nested_preview/necromancer_readout_nested.html) -
  prose-regex "produces vs. modifies" classification with family-of-spells
  header folding. See `Docs/PIPELINE.md` section 8. Superseded in spirit
  by the DBC-sourced model above, kept as a separate experiment since it
  covers a different detection mechanism (verb-adjacency in tooltip text,
  not spellId links).

---

Regenerating any of these overwrites the file in `Outputs/` in place - the
links above always point at the current version. See `Docs/PIPELINE.md`
for the full regeneration commands per file.
